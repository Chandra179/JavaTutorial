Be careful not to write accessor methods that return references to mutable objects


As a rule of thumb, always use clone whenever you need to return a copy of a mutable field.

If you define a field as static, then the field is not present in the objects of the class. 
There is only a single copy of each static field.


The static keyword in Java is mainly used for memory management. 
The static keyword in Java is used to share the same variable or method of a given class. 


static methods vs factory methods


There are, however, two kinds of method parameters:
  • Primitive types (numbers, boolean values)
  • Object references


You have seen that it is impossible for a method to change a primitive type parameter. 
The situation is different for object parameters. 
You can easily implement a method that triples the salary of an employee:

  public static void tripleSalary(Employee x) // works
  {
     x.raiseSalary(200);
  }
